DNS-запрос (команда nslookup или dig)
| Как проверить, работает ли этап: проверить ответ от nslookup или dig
| Какие инструменты использовать для диагностики: с помощью команды nslookup/dig имя_домена.доменная_зона
| Какие проблемы могут быть: может выдаваться неверный IP адрес / не получим IP адрес сервера
| Как решить проблему: проблема может быть в неправильной конфигурации dns (ее нужно проверить) / использовать другой
DNS-сервер


TCP-соединение (команда ping или telnet)
| Как проверить, работает ли этап: командой ping ip_address / открыв ресурс в браузере (если не отображается проблема
может быть НЕ в tcp)
| Какие инструменты использовать для диагностики: ping / telnet
| Какие проблемы могут быть: перегруженность сервера / недоступность / сервер выключен / закрыт порт
| Как решить проблему: открыть порт / проверить сеть / настроить работу веб сервера nginx


HTTP-запрос (инструменты разработчика браузера, curl)
| Как проверить, работает ли этап: проверка корректности отправки запроса / проверка статуса ответа
| Какие инструменты использовать для диагностики: curl команда / devtools в браузере вкладка Network посмотреть все
отправленные запросы -> полученные ответы
| Какие проблемы могут быть: неправильный формат запроса (метод запроса например) / ошибки в headers / редиректы, tls
nginx /
| Как решить проблему: настроить nginx прокси, редиректы, tls / проверить все параметры обработчика пути (заголовки /
путь / метод и тд)


Обработка на сервере (логи сервера, время ответа)
| Как проверить, работает ли этап: проверить приходит ли запрос на сервер и сколько времени занимает обработка
| Какие инструменты использовать для диагностики: логи сервера (nginx) / логи приложения /
команда curl с измерением времени ответа
| Какие проблемы могут быть: ошибка в коде приложения / медленные SQL запросы / блокировки в базе данных /
перегруженность CPU или памяти / зависшие процессы
| Как решить проблему: посмотреть стек ошибки в логах / оптимизировать SQL запросы / проверить блокировки в базе /
увеличить ресурсы сервера / исправить баг в коде приложения


HTTP-ответ (размер ответа, статус)
| Как проверить, работает ли этап: проверить статус HTTP ответа и размер возвращаемых данных
| Какие инструменты использовать для диагностики: curl / devtools Network
| Какие проблемы могут быть: сервер возвращает ошибки (500 / 404 / 502) / слишком большой размер ответа /
медленная передача данных / неправильные headers
| Как решить проблему: исправить ошибку на сервере / оптимизировать размер ответа (сжатие gzip или brotli) /
проверить конфигурацию nginx / исправить обработчик запроса


Рендеринг (инструменты разработчика браузера)
| Как проверить, работает ли этап: страница загрузилась но долго отображается или отображается некорректно
| Какие инструменты использовать для диагностики: devtools браузера вкладки Network / Performance / Console
| Какие проблемы могут быть: большие js файлы / блокирующие скрипты / ошибки javascript / медленная загрузка ресурсов
(картинки / медиафайлы / css / js)
| Как решить проблему: использовать lazy loading ресурсов / оптимизировать изображения /
исправить ошибки javascript / пагинация данных
