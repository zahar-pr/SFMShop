# Требования SFMShop (интернет-магазин):
### связи между таблицами JOIN Relationship (Пользователи -> заказы -> логистика и тд.)
### соблюдение принципов ACID, включает в себя: корректная обработка параллельных транзакций, атомарность транзакций, консистентность данных (нельзя создать заказ для несуществующего пользователя). Всё это критически важно для интернет-магазина

### быстрый поиск по индексам

# PostgreSQL vs MongoDB

### 1. PostgreSQL
В интернет-магазине есть строгие связи между таблицами:

- Пользователь (users)
- Заказы (orders)
- Товары (products)
- Доставка (shipping)

Например:

- один пользователь может иметь много заказов
- один заказ может содержать много товаров
- товар может входить в много заказов 

Это реализуется в **реляционной БД**.

В PostgreSQL это реализуется через:

- `FOREIGN KEY`
- `JOIN`
- ограничения целостности

Пример:

`orders.user_id -> users.id`
`order_items.order_id -> orders.id`
`order_items.product_id -> products.id`

### 2. MongoDB
Там где не нужна честкая структура данных и сами данные НЕ чувствительны.
Например: Логи приложения, аналитика кликов и другая не чувствительная аналитика, метрики, данные без четкой 
структуры, настройки пользователя (пример - настройки клиентского UI которые будут храниться на сервере)
