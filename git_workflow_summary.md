# Git Workflow для проекта SFMShop

## Использованные команды Git

- **git init** — инициализация локального Git-репозитория.
- **git add** — добавление файлов в staging area перед созданием коммита.
- **git commit** — фиксация изменений в истории репозитория.
- **git log** — просмотр истории коммитов.
- **git log --oneline --graph --all** — компактная визуализация истории коммитов со структурой веток.
- **git log --stat** — просмотр статистики изменений по файлам в коммитах.
- **git branch** — просмотр и управление ветками.
- **git checkout** — переключение между ветками или коммитами.
- **git diff** — просмотр различий между ветками, коммитами или файлами.
- **git show** — просмотр содержимого конкретного коммита.
- **git push** — отправка локальных изменений в удалённый репозиторий.
- **git pull** — получение изменений из удалённого репозитория.
- **git merge** — объединение изменений из одной ветки в другую.

# Созданные ветки

В проекте используется простая стратегия ветвления, основанная на **master + feature branches**.

### Основная ветка

- **master** — основная ветка проекта.  
  В ней хранится стабильная версия кода. Все feature-ветки в конечном итоге сливаются в неё после завершения разработки.

### Feature-ветки

Feature-ветки используются для разработки отдельных функциональных возможностей без риска сломать основную ветку.
